基于Hexo+GitHub pages的博客搭建

“Yeah It’s on. ”

我的Blog 就这么开通了。

跳过废话，直接看技术实现

搭建博客需要的东西

  • 一个GitHub账号 传送门<–
  • git for Windows 传送门<–
  • node.js 传送门<–
  • hexo 传送门<–

一 .安装Node.js

  • 点击上面的链接访问Node.js官网，下载长期支持版，双击安装下载下来的文件。没什么好说的，一路下一步，选项都是默认的。

    二.安装Git

  • 点击上面的链接访问Git的官网，下载相应的版本，默认安装即可

  • 用win+r运行cmd，依次输入

git --version 
node -v  
npm -v

  • 如果出现的版本信息，即安装成功


    三.安装Hexo

  • 选择一个地方，新建一个文件夹，博客相关的文件将会存储在此文件夹中，然后在该文件夹下，点击鼠标右键，，打开 Git Bash Here ，依次输入下列命令
    npm install hexo-cli -g
    npm install hexo-deployer-git --save

  • 安装完成后，输入 hexo --v 出现版本信息则安装成功


    四.生成Hexo静态网页

  • 在当前文件夹继续新建一个文件，进入文件夹，右键鼠标，打开 Git Bash Here ，输入 hexo init 等待下载，hexo将会下载一些工程文件到这个目录中去，整个下载过程可能比较漫长，请耐心等待

  • 下载完毕后，在当前目录中将会多了许多文件，接着，我们运行 hexo g 生成网站的静态网页文件,命令运行完毕后，再运行 hexo s 使这个网站在本地先运行起来，如果出现 Hexo is running at... 我们在浏览器中打开 http://localhost:4000



  • 到目前为止，我们的本地博客就成功搭建了，但是现在我们只能通过本地连接查看博客，我们要做的是让其他人也能够访问我们的博客，这就需要我们将博客部署到Github Pages上

    五.配置GitHub

  • 用创建好的账号登录github，并创建一个仓库，仓库名为xxx.github.io，xxx为你的用户名，然后 Creat repository，因为我已经创建了这个仓库，所以出现了警告

  • 仓库建好后，我们将使用git来获取SSH

  • 点击鼠标右键，选择Git Bash Here，在Git Bash中输入：ssh-keygen -t rsa -C "你的邮箱地址"，接着，根据提示，你需要输入密码和确认密码（可以不用密码，就是到输密码的地方，都直接回车，所以每次push就只管回车就行了。）。此时，生成密钥后，在你电脑目录：C:\Users\你的计算机用户名.ssh 下，会生成两个文件


  • 私钥：id_rsa

  • 公钥：id_rsa.pub

  • 用记事本打开 id_rsa.pub ,复制里面的内容，然后访问 https://github.com/settings/ssh 点击右上角的 New SSH key ,出现如下界面

  • Title 自己随便取，Key 为刚刚复制的内容

  • 现在你已经可以通过 SSH 链接到 GitHub 了，还有一些个人信息需要完善的。Git 会根据用户的名字和邮箱来记录提交。GitHub 也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的。GitHub 就配置完了

    git config --global user.name "your user.name"

    git config --global user.email "your user.email"
